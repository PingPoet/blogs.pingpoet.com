VS2005 Doesn't Do Much For C# XML Documentation

I am disappointed that VS2005 Beta1 has no improvements around using XML doc include. I have my team using the include now on a new project with a UE team and I'm tempted to dump it because it limits some functionality and is such a hassle to use. Firstly using include files doesn't catch as many doc warnings as it would if the comments were in the source which I think it very important. For example if you use a <param... tag with an invalid name or have a missing one you get no warnings from the compiler/IDE when rebuilding, which you would get the helpful warnings if the comments are the source file (you know where it says "Parameter xxx' has no matching param tag in XML comment (but other parameters do)". So instead of getting warning from the compiler each time you build (like you normally would if comments were inline in the code) you just end up with a bad documentation file. Secondly a tool something like what Issam mentioned in EricGu's post is desperately needed in order to efficiently use XML doc includes. Since the rumor is Microsoft uses these for their own documentation internally, my bet is that have an internal tool they're not sharing with us.

The call to Server.CreateObject failed while checking permissions...

Receiving “The call to Server.CreateObject failed while checking permissions. Access is denied to this object.” When calling CreateObject within an ASP application? Simply grant the web user’s Anonymous access account (and any application pool account if IIS6) permissions to read and execute the .dll file containing the COM object you’re trying to create with the CreateObject function.
