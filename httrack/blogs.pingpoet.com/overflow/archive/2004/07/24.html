Automated Backup for StarTeam + MSSQL (with full Delphi source-code)

This might come in handy...

Have you ever questioned why is it that StarTeam doesn't come with an automated backup tool? There's a good answer for that, although I haven't got official corroboration for it: StarTeam supports several database server backends and this tool would have to deal with the specific backup procedures for each of them. That's my opinion: DSP BU folks, please correct me if I am wrong :)

Before going on, let's take a look at the supported/official StarTeam backup procedure:

    1. Lock the server  (optional)
    2. 20-30min after locking, shut down the StarTeam server
      • command-line: starteamserver -stop
    3. Backup
      §         Database (using proper backup software) or SQL Server raw database files (mdf, ldf & ndf files)
      §   \StarbaseServer.ini
      (StarTeam 5.x)
      §   \starteam-server-configs.xml
      (StarTeam 6.x)
      §   Repository Folder
    4. Restart StarTeam
      • starteamserver –start

Those are the necessary steps in order to make sure that the database is properly synchronized with the repository (that resides in the filesystem).

But lo! Here is a small tool we've developed here at Borland Latin America PSO with full Delphi 7 (what else ) source code that will automate the whole process above when MSSQL is the backend database:

http://codecentral.borland.com/codecentral/ccWeb.exe/listing?id=21897

Here's its help screen:

Borland StarTeam 6.0 + MS SQL Backup Utility v1.0.0.1
Copyright (c) 2004 Borland Latin America

  Usage:
    stbackup_mssql.exe
                      
                       [-subdirs]

    Options:
      subdirs (optional) = creates a subdirectory in the output path
                           in the following format: yyyy-mm-dd-hh-nn-ss

  Example:
    stbackup_mssql StarDraw c:\Backup (local)\StarTeam sa sa_pwd

Have fun! (no other reason for work, is there?) :)

DISCLAIMER: This is not an officially Borland supported tool. Although it was tested in-house and in customers, use it at yout own risk. If you ever need any further clarification regarding this tool, please add a comment to this blog entry.
