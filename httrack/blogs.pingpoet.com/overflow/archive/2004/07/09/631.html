Handling Native Unhandled Exceptions on Windows

Recently I was asked by a friend what he could do about a native exception that occurred when he closed a managed WinForms application and he could not catch with a try/catch. The exception appeared to be occurring in the COM marshaling or the interop assembly that was used to host an ActiveX/COM control from the windows forms application. Since the error was not a managed exception it was pretty nasty issue to deal with. On top of that the end user got what he wanted out of the application, everything had already worked, he just gets an ugly message when trying to close it. According to Chris Brumme on Unhandled Exceptions, this should not happen under most circumstances, but unmanaged components can register to "handle" native exceptions, and I do know that the COM component being used was a VC6 component, so the VC6 filter he mentions may be the culprit. Anyway, to fulfill the quest for an easy way out of the problem, I suggested using the Windows API to register a native exception handler and ignore it when closing the application. The windows API needed is SetUnhandledExceptionFilter. (The "Send This Error Report to Microsoft" dialog is implemented doing something like this.) Below are some links to some articles that might be helpful for those trying to use it:

  • http://pinvoke.net/default.aspx/kernel32.SetUnhandledExceptionFilter
  • http://www.ftponline.com/archives/listissue.asp?pubID=1&MagIssueId=186# (See the article titled "Black Belt Programming: No Exception Errors, My Dear Dr. Watson" and the link to the code.)
  • http://discuss.fogcreek.com/joelonsoftware/default.asp?cmd=show&ixPost=72832
  • Fault Protection: Using Windows Exception Filters
  • Emulate VB.NET Error Handling
  • http://msdn.microsoft.com/library/en-us/debug/base/setunhandledexceptionfilter.asp

Note that calling a native Windows API is only necessary because this unhandled exception manifested itself as a "native" or "unmanaged" exception. If the error manifested itself as a managed unhandled exception then the AppDomain.UnhandledException Event can be used.
