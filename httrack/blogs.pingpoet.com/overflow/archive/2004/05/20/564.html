WHIRL Away

Miguel points out that Kristian Rietveld is making progress on a WHIRL-to-IL project. WHIRL is an intermediate language used by the Open64 Compiler Tools. Apparently some modified GCC compilers for the C, C++ and Fortran languages already support WHIRL and Kristian's work will make it possible to get these languages to work under mono or other CLI runtime environments. Very interesting projects, I can't wait to see them.

On a similar note, I'd be even more interested in seeing an IL-to-WHIRL converter and here's why. The big difference between the WHIRL system and the IL/CLI system is that CLI depends on a fairly large runtime environment being present on the destination machine where as WHIRL actually gets compiled directly to native machine code for the target computer. Native code in many cases yields better performance, better memory usage, and a more predictable memory usage and collection pattern, but most importantly native code can yield a MUCH SMALLER distribution for an application. In many cases requiring a customer to download and install something like the mono, the Microsoft .NET Runtime or the Java JRE, is not desirable for some applications; particularly consumer applications that are mostly distributed by the Internet. Many of us geeks might think "everyone has .NET installed already", but I assure you that is far from the truth. And I know some of you are thinking since Longhorn will come with .NET installed this means even more people will have .NET installed, but if you think everyone will just run out and install Longhorn the moment Microsoft releases it (in 2006) I think you're in for a surprise. Downloading 20MB might not seem like a big deal at first, after all it's only 20MB, but allot of consumer applications and utilities are a mere fraction of that size so requiring and slight additional download that is 4-5 times larger is just not acceptable. Of course this is nothing a good ol' Linker couldn't fix either.
