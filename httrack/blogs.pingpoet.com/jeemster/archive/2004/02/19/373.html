SMTP servers and Blocked sites.

SMTP servers and Blocked sites.

 Well some one said they tried to send me mail and it never got to my mail box. On some investigation, we find out my SMTP server is blocking as they do not have a revers DNS entry on their Ip address. On checking my logs, I see there have been, well, several blocked hosts.

So, I go to http://www.dnsreport.com/ and put in their domain name and the report says:

FAIL Reverse DNS entries for MX records

ERROR: The IP of one or more of your mail server(s) have no reverse DNS (PTR) entries (if you see "Timeout" below, it may mean that your DNS servers did not respond fast enough). RFC1912 2.1 says you should have a reverse DNS for all your mail servers. It is strongly urged that you have them, as many mailservers will not accept mail from mailservers with no reverse DNS entry. You can double-check using the 'Reverse DNS Lookup' tool at the DNSstuff site.
The problem MX records are: ###.###.###.###.in-addr.arpa [No reverse DNS entry (rcode: 3 ancount: 0)]

So, do we or don't we want to block these SMTP hosts ?

Well a quick Google search shows many people are blocking servers with no revers DNS to fight spam.

Then the latest attempt to slow spam, some are blocking any SMTP servers that are in a designated DHCP space. Some mention they even check that the reverse DNS contains the something indicating the forward DNS host. (I think this is reaching a little.)

But generally, the consensous is that hosts with no reverse DNS address should be blocked.

So, I bought my own doman, okay several domains, all for my personal use, and run a little SMTP server in my garage. It has a reverse DNS entry, but, I have been unable to send mail from my SMTP server as too many sites block the revers DNS enrty as it has "DHCP" in the reverse entry.

What's a guy to do ?
