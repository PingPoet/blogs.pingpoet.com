Why aren't my aspx pages loading under IIS?

Twice while working with Asp.NET and IIS I’ve had problems with aspx pages not loading when trying navigate to them in the browser. Instead showing the aspx page the browser would give 404 errors. For both cases IIS appeared to be setup with all the correct Asp.NET mappings and other non Asp.NET pages in the same site would load fine.

The reason I decided to post this was because each time the problem occurred there was a different cause and I spent more than a few hours trying to figure out what was going on for each one.

Windows XP Pro

The first time I had the problem was with Windows XP pro on my dev machine. In order to fix the problem I had to completely remove all Asp.NET mappings from IIS and recreate them. If you’re reading this you are probably already aware of the aspnet_regiis.exe –i solution posted all over the web that is supposed to fix the problems but it didn’t. In order to fix the problem I had to do the following:

Remove all mappings for Asp.NET from IIS by using the following commands:

aspnet_regiis.exe –ua
aspnet_regiis.exe –k W3SVC/1/Root
aspnet_regiis.exe –ea

After running these commands I recommend restarting IIS for good measure.

Double check to make sure all mappings are really removed by using the following commands:

aspnet_regiis.exe –lv
aspnet_regiis.exe –lk

Once everything is cleared out setup the mappings again using the following commands:

aspnet_regiis.exe –i
aspnet_regiis.exe –r
aspnet_regiis.exe –c

Please note this is a brute force approach and will switch all the maps to the version of Asp.NET you are running aspnet_regiis.exe from.

Windows 2003 Server

The second time the problem occurred was on a windows 2003 standard server. I spent a long time trying to figure out why the above approach wasn’t fixing the problem when I finally I discovered IIS 6.0 actually prohibits Asp.NET by default. To allow Asp.NET to work do the following:

  1. Go into the IIS manager console.
  2. Find the Web Service Extensions node in the tree view and click on it.
  3. In the panel on the right you should see a list of web service extensions.
  4. Find the Asp.NET extension and check to see if its status is prohibited.
  5. If it is, click on the Asp.NET extension and click the “allow" button.
Alternatively, you can use the apsnet_regiis.exe –i –enable command to enable this as well.
