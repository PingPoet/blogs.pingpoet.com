How you show a mbUnit report in the CruiseControl for .NET Web Dashboard

I recently needed to update our build process to show our mbUnit reports on the CruiseControl for .NET Web dashboard. Trying to get all the pieces together to figure out how to do this was a scavenger hunt of sorts so I decided to put all the information I gathered together in one place. This is the very basics of what I found and fairly simple to implement right away. I'm sure there are more advanced ways of doing this but hopefully this information will get you going fairly quickly.

  1. Acquire the xsl file
    • Since mbUnit is now being bundled inside the TestDriven.NET project, download the setup from http://www.testdriven.net. The xsl file can be found in the TestDriven.NET install under the following folder: ..\TestDriven.NET 1.0\MbUnit\CruiseControl

  2. Create plugin for the Dashboard
    • Make sure you have the latest source of CrusiseControl for .NET. At the time of this writing the version is 0.8
    • Copy the xsl file to ..\CruiseControl.NET-0.8.source\project\xsl (I renamed it to mbUnit.xsl)
    • Open up the solution file ..\CruiseControl.NET-0.8.source\project\ccnet.sln
    • Navigate to the Webdashboard project's xsl folder and add the xsl file you just copied over
    • Navigate to the Webdashboard project's Plugins folder
    • Add a folder for mbUnit
    • Add a code file to the folder call mbUnitTesResultsPlugin.cs
    • Copy/Paste the following code in to the code file
      using ThoughtWorks.CruiseControl.WebDashboard.Dashboard;
namespace ThoughtWorks.CruiseControl.WebDashboard.Plugins.mbUnit
{
    public class mbUnitTestResultsPlugin : XslReportBuildPlugin
    {
        public mbUnitTestResultsPlugin(IBuildLogTransformer buildLogTransformer)
            : base (buildLogTransformer, @"xsl\mbUnit.xsl", "View mbUnit Report", "ViewmbUnitReport")
        { }
    }
}
    • Compile the project

  3. Deploy
    • Copy the newly compiled Webdashboard assembly to the deployment machine's ..\webdashboard\bin folder
    • copy the xsl file to the deployment machine's ..\webdashboard\xsl folder
    • Added the new plungin and xsl information to the web.config file
    <BUILDPLUGINS>
...
<PLUGIN typeName="ThoughtWorks.CruiseControl.WebDashboard.Plugins.mbUnit.mbUnitTestResultsPlugin" />
</BUILDPLUGINS>
<XSLFILES>
...
    <FILE name="xsl\mbUnit.xsl" />
</XSLFILES>

  4. Putting it all together
    • To make it all work make sure your build script is running mbUnit and generating an xml report
    • In the CruiseControls server config file merge the mbUnit xml report
    <TASKS>
    <MERGE>
        <FILES>
            ...
            <FILE>..\results\mbUnit-report.xml</FILE>
        </FILES>
    </MERGE>
</TASKS>
