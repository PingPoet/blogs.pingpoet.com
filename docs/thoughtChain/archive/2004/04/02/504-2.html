ActiveReports vs MS Reporting Services

Adnan Masoud is looking for some benchmarks between AR and MSRS.  Here is my response to help Adnan and others decide on which product to use.

First, I'll start by saying I am the product manager for ActiveReports and I am also responsible for our development and marketing strategy in response to MSRS.   Our strategy in one word: “Trial”.  We offer a full evaluation of ActiveReports on our web site www.datadynamics.com and MSRS is available on Microsoft's web site.  Anyone who is looking at a reporting solution should download both and try to create a few reports with each and make the decision based on that experience.

I believe MSRS has a slightly different target from ActiveReports.  Our users tend to want more control over every aspect of the report and that's what we provide.   MSRS provides an excellent set of features and a really good architecture, however, some developers might find it constraining in certain areas.

ActiveReports is a component that integrates as part of the application, it does not have any stand alone components.  The reports themselves end up as classes just like any other class in your code.  You can add properties, code events and have complete control over their execution.  The same model works in Windows and Web applications.

Code behind and report events are great features of AR.  They are difficult to justify on first look, however, they are critical when you need them.  I've seen many of users perform things that are impossible in other report designers because of the ability to write code in their language of choice for the report.

Story :) - We have a customer that wanted to print a set of monthly statements (>1000 pages)  and needed to export these statements (individually) to PDF  based on the users request.  He ended up generating the report output then taking the individual pages (using our RDF file format) and inserting each page as a record in a database with some keywords.   Now his users can use his app to pick and choose which of those pages they need.  The app would then grab the selected pages from the database, merge them into a new RDF document and export the document to PDF (or any other format) as if it was run that instant.   You simply cannot do that in any other report writer.

ActiveReports is very small and less complicated than MSRS, no-hassle deployment is a key factor that established AR against Crystal and I think it will do the same against MSRS.  The engine is an assembly that you can deploy with XCopy.   MSRS installation is a process that can be difficult at times.

Pricing is another hidden part of this comparison.  MSRS starts off as free, however, as soon as you choose to deploy the reporting server separate from your SQL server, costs will ramp up very very quickly because each RS instance would require a separate SS2K license.

Features

I think MSRS has great set of controls (List, Table, Chart and Matrix) that make reporting very easy.  We are working on adding a Table and Chart controls to AR and will eventually add the Matrix control.  We are also adding support for multiple data sources, multiple detail sections (nested and sequential). 

However, I think that AR with its flexible architecture and application integration is the right developers' choice.

Bottom line, try both, see what fits your needs and go with it.  That's what I would do.  And if you have any questions about AR, leave me a note here or send us an email.
